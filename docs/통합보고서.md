## 단계별 상세

### 1. 영상 수집 (완료)

- 경동고속도로 23개 IC CCTV 영상
- 총 2,581개 AVI, 409GB
- 원본: `/mnt/Expansion/영상/220603_고속도로(경동)/`

### 2. 검출 + 추적 (완료)

- 차량 검출: YOLO11n (COCO pretrained)
- 차량 추적: ByteTrack
- 영상 종료 시 차량별 최고 품질 크롭 3장 선별 → 일괄 분류

### 3. 분류 모델 개발 (완료)

- 7종 분류: T1(승용), T2(버스), T3(소형화물), T4(중형), T5(대형), T10(세미트레일러), T13(이륜)
- 15 라운드 실험 → Triple Ensemble V2 확정
- 학습 데이터: 자체 경동 924장 + AI Hub #165 2,542장 = 3,466장

### 4. 봉인 평가 (완료)

- 경동 테스트셋: 175장 (SHA256 봉인)
- 시내 테스트셋: 150장 (SHA256 봉인)
- Frame 정확도 64.6% / 77.3%
- **Vehicle 단위 집계 시 T1 오차 ±0.1%p → 실무 수용 가능**

### 5~7. 크롭 추출 + 라벨링 준비 (완료, 04-27)

**목적**: 분류 모델 성능 향상을 위한 추가 학습 데이터 확보

- 50영상 (야간 30 + 역광 12 + 주간 8)에서 **76,515 크롭** 추출
- 이미지+메타데이터를 Parquet 1개 파일로 통합 (1.3GB)
- 그중 **1,000장**을 50장씩 20 배치로 분할

### 8. Claude 사전 라벨링 (미착수)

**방법**: Claude가 크롭 이미지를 보고 차종 사전 분류 (subagent 위임)

- 배치당 50장 × 20 batches = 1,000장
- 차종별 GT 앵커 이미지 3장씩 참조 (few-shot)
- 신뢰도 < 0.7은 재검토 플래그
- 예상 소요: ~30분

### 9. 사용자 OX 검수 (대기)

- Claude 라벨 결과를 사용자가 수용(O) / 거부(X) 판정
- 장당 0.5~1초 (기존 직접 라벨링 대비 4~8배 빠름)
- GUI 도구 준비 완료 (`crop_review_gui.py`)

### 10. 분류기 재학습 — R16 (대기)

- 검수 완료 라벨을 학습 데이터에 추가
- 목표: 자체 경동 라벨 924 → 1,883장 (AI Hub 비율 균형 맞춤)
- 주요 보강 대상: T2(버스) +297, T5(대형) +216, T3(소형화물) +216

### 11. 전체 재처리 (대기)

- R16 모델로 23 IC 2,581 영상 전체 재처리
- IC별 차종 분포 CSV + 집계 JSON 생성

### 12. 납품 리포트 (대기)

- Vehicle-level 교통량 오차 검증
- IC별 차종 분포표
- Bootstrap 95% 신뢰구간
