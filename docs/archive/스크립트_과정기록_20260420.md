## 2. 카테고리별 상세 과정 기록

### 2.1 데이터 준비 (임베딩 + 클러스터링)

#### `extract_embeddings.py`

| 항목 | 내용 |
|------|------|
| **목적** | 147K crop 이미지의 시각 특징 벡터를 추출하여 후속 클러스터링의 입력으로 사용 |
| **방법** | DINOv2-small(facebook/dinov2-small) 모델의 CLS 토큰(384d)을 배치 추론으로 추출. PIL -> AutoImageProcessor -> AutoModel -> last_hidden_state[:,0,:] |
| **파라미터** | model=`facebook/dinov2-small`, input=`verified_clean/{C1~C6}/*.jpg`, batch_size=32, output_dim=384, device=cpu |
| **결과** | `/workspace/prj_cctv/output/embeddings.npz` (201MB, paths+labels+embeddings) |
| **판단근거** | DINOv2-small은 라벨 없이 시각 특징을 잘 추출하며, 384d는 클러스터링에 적합한 차원. GPU 없이 CPU 추론 가능. |

#### `cluster_embeddings.py`

| 항목 | 내용 |
|------|------|
| **목적** | DINOv2 임베딩을 클러스터링하여 차종별 그룹 자동 분리 |
| **방법** | L2 정규화 -> UMAP(384->32d) -> K-Means(K=50) -> 클러스터별 그리드 이미지 생성 |
| **파라미터** | UMAP: n_components=32, n_neighbors=30, min_dist=0.0, random_state=42. K-Means: n_clusters=50, n_init=10. 그리드: cell=100px, cols=8, samples_per_cluster=24 |
| **결과** | `clusters/cluster_assignments.npz`, `clusters/grids/cluster_00~49.jpg` |
| **판단근거** | K=50은 6종 원본 대비 세분화하여 대형화물 내 트레일러/단차 분리를 시도하기 위한 설정. UMAP으로 차원 축소하여 K-Means 성능 향상. |

#### `recluster_k200.py`

| 항목 | 내용 |
|------|------|
| **목적** | K=50에서 분리되지 않은 혼합 클러스터를 세분화하고, 소형 bbox 노이즈를 자동 제거 |
| **방법** | bbox area<8000 사전 NOISE(-1) 마킹 -> 양호 데이터만 UMAP+K-Means(K=200) |
| **파라미터** | K=200, MIN_AREA=8000, UMAP: 32d/n_neighbors=30. 그리드: cell=100, cols=8, samples=24 |
| **결과** | `clusters_v2/cluster_assignments_v2.npz`, 200개 그리드 이미지 |
| **판단근거** | K=200은 C5(대형화물) 클러스터를 트레일러(T10)/세미(T8)/단차(T5)/덤프 등으로 세분화하기 위해 설정. MIN_AREA=8000은 실측 기반으로 유의미한 차량 crop의 최소 크기. |

#### `subcluster_c33.py`

| 항목 | 내용 |
|------|------|
| **목적** | C33(대형화물 8178장) 내에서 트레일러(8~12종) vs 단차 대형(5~7종) 분리 |
| **방법** | C33 멤버만 추출 -> L2 정규화 -> K-Means(K=15) -> 그리드 생성 |
| **파라미터** | K=15, input: cluster_id==33인 8178장 |
| **결과** | `clusters/c33_sub/sub_assignments.npz`, 15개 그리드 이미지 |
| **판단근거** | C33은 K=50에서 대형 화물차가 집중된 클러스터. K=15로 세분화하여 트레일러 유형별 분리 시도. Claude 시각 검토로 sub별 라벨 할당. |

### 2.2 라벨링 및 정제

#### `apply_cluster_labels.py`

| 항목 | 내용 |
|------|------|
| **목적** | Claude 시각 검토 결과를 50개 클러스터에 적용하여 147K 이미지에 13종 라벨 전파 |
| **방법** | CLUSTER_LABELS 딕셔너리(50개 클러스터->13종 매핑) 하드코딩 -> 전체 이미지에 적용 |
| **파라미터** | 매핑: T1(20개 클러스터), T2(4개), T3(1개), T4(3개), T5(1개), NOISE(18개), MIXED(2개) |
| **결과** | `clustered_labels.csv` (path, original_6cls, cluster_id, label_13cls, label_name, is_noise) |
| **판단근거** | Claude가 그리드 이미지를 보고 각 클러스터의 주요 차종을 판별. NOISE 클러스터(트래킹 오류, 고정 시설물)를 적극 분리. |

#### `auto_label_v2.py`

| 항목 | 내용 |
|------|------|
| **목적** | K=200 클러스터에 응집도 기반 자동 라벨 부여 |
| **방법** | 클러스터별 원본 6종 분포 계산 -> 응집도 >= 80%: AUTO, 50~80%: REVIEW, <50%: MIXED |
| **파라미터** | AUTO 임계값=0.80, REVIEW 임계값=0.50. 6종->13종 기본 매핑: C1->T1, C2->T2, C3->T3, C4->T4, C5->T5, C6->T13 |
| **결과** | `cluster_v2_labels.csv` (cluster_id, n, dominant_6cls, purity, auto_label_13cls, status, dist) |
| **판단근거** | 응집도 80%+ 클러스터는 자동 라벨이 안전. REVIEW/MIXED는 수동 검토 필요. |

#### `finalize_labels.py`

| 항목 | 내용 |
|------|------|
| **목적** | K=50 클러스터 + C33 서브클러스터 결과를 통합하여 최종 13종 라벨 생성 |
| **방법** | CLUSTER_LABELS(50개) + SUB33_LABELS(15개) 매핑 -> CSV + verified_13cls/ 심링크 디렉토리 |
| **파라미터** | SUB33 매핑: 0,1->T10, 2,3,4,12->T4, 5->T8, 9->T5, 10->T6, 11->T3, 7,8,14->NOISE, 6,13->MIXED |
| **결과** | `final_labels.csv`, `labels/verified_13cls/{종별}/*.jpg` (심링크) |
| **판단근거** | 50개 클러스터 라벨 + C33 세분화 라벨을 병합하여 최대한 세밀한 13종 분류 달성. |

#### `finalize_v2.py`

| 항목 | 내용 |
|------|------|
| **목적** | K=200 기반 최종 라벨 v2 생성 (수동 보정 포함) |
| **방법** | 응집도 기반 자동 라벨 + MANUAL_LABELS(Claude 검토한 C5 클러스터 8개) 결합 |
| **파라미터** | MANUAL_LABELS: {0:T10, 94:T10, 68:T5, 171:T5, 145:T8, 198:T5, 36:T5, 128:T4}. 작은 bbox(cluster_id<0)->NOISE_size |
| **결과** | `final_labels_v2.csv` (path, original_6cls, cluster_id, label_13cls, label_name) |
| **판단근거** | C5 계열 클러스터를 Claude가 시각 검토하여 T5/T8/T10 수동 라벨 부여. 자동+수동 하이브리드로 정확도 향상. |

#### `confident_learning.py`

| 항목 | 내용 |
|------|------|
| **목적** | KNN + cleanlab으로 노이즈 라벨을 자동 식별하고 정제 |
| **방법** | v2 라벨 + DINOv2 임베딩 -> L2 정규화 -> KNN(k=20, cosine) cross-val -> cleanlab find_label_issues -> 3단계 정제(KEEP/RELABEL/NOISE) |
| **파라미터** | KNN: n_neighbors=20, metric=cosine, cv=3. cleanlab: filter_by=prune_by_noise_rate. RELABEL 조건: issue + KNN conf>0.7 + 라벨 불일치 |
| **결과** | `final_labels_v3_clean.csv` (path, v2_label, knn_label, confidence, is_issue, action, final_label) |
| **판단근거** | Confident Learning(JAIR 2021)은 학술적으로 검증된 노이즈 라벨 탐지 기법. KNN은 시각적 유사성 기반으로 라벨 일관성을 검증. |

#### `manual_correction.py`

| 항목 | 내용 |
|------|------|
| **목적** | GT 시각 검증 결과를 반영한 수동 보정 (v3->v4) |
| **방법** | 규칙 기반 보정: T3->T1(90% 승용차 판명), T8->T4(0% 정확도), T5는 area>=80000만 유지(미만은 T4) |
| **파라미터** | T5_MIN_AREA=80000. 최종 6종: T1, T2, T4, T5, T10, T13 |
| **결과** | `final_labels_v4.csv` (v4_label, v4_name 컬럼 추가) |
| **판단근거** | GT 500장 시각 검증에서 T3(포터) 90%가 승용차로 판명. T8은 트레일러 분리 실패(0% 정확). T5는 면적 기반 분리가 유효. |

#### `self_training.py`

| 항목 | 내용 |
|------|------|
| **목적** | v4 모델로 전체 데이터를 재예측하여 v5 라벨 생성 (self-training) |
| **방법** | cls_v4_best.pt로 147K장 배치 추론 -> conf>=0.7: 모델 라벨 우선(MODEL/AGREE), conf<0.7: NOISE_lowconf |
| **파라미터** | model=cls_v4_best.pt, BATCH=256, imgsz=128, HIGH_CONF=0.7. NOISE는 그대로 보존(NOISE_KEEP) |

| **판단근거** | self-training은 고신뢰 예측을 정답으로 재활용하는 반지도학습 기법. v4-v5 일치율 98.2%로 모델 안정성 확인. |

## 3. 파이프라인 단계별 스크립트 매핑

```
Phase 0: 검출 검증
  -> (외부 모듈: classifier/detector.py)

Phase 1: 트래킹 + 카운팅
  -> tracker.py
  -> run_batch_all_crops.py (2581영상 crop 수집)

Phase 2: 6종 분류기 fine-tune
  -> extract_embeddings.py (DINOv2 임베딩)
  -> cluster_embeddings.py (K=50)
  -> recluster_k200.py (K=200)
  -> subcluster_c33.py (C33 세분화)
  -> apply_cluster_labels.py / auto_label_v2.py (라벨 할당)
  -> finalize_labels.py / finalize_v2.py (통합)
  -> confident_learning.py (노이즈 정제)
  -> manual_correction.py (수동 보정)
  -> self_training.py (재라벨)
  -> build_dataset_v4.py / v5.py (데이터셋)
  -> train_v4.py / train_v5.py (학습)
```
