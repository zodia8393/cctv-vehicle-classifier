## 5. 파이프라인 v2 개선 (Lazy Classification)

### 5.1 기존 v1 문제

- `classify_every_n=10`: **고속 합류부 차량 95% 누락** (track span 12-21 frames)
- Tracker 다수결 필요 → 짧은 track은 구조적 탈락

### 5.2 신규 v2 (5 스킬 회의 합의)

```
┌── 전 프레임 detector + tracker
├── track별 crop buffer에 quality score 기록
│     quality = area × det_conf × center_proximity
├── Virtual Line Crossing 기록 (FHWA/VDS 표준)
└── 영상 종료 시점:
     ├── 필터: min_track_frames≥6 + min_trajectory≥80px
     ├── best-N (=3) crops 품질 상위 선별
     ├── batch classify (한 번의 forward)
     └── conf-weighted majority vote → final_class
```

### 5.3 효과 (hiv00179 합류부)

| 지표 | v1 (classify_every_n=10) | **v2 (Lazy)** |
|------|---:|---:|
| valid tracks | 5 | **177** |
| 개선 배수 | — | **35×** |
| 분류 호출 | N 프레임마다 | **track당 3회** (-99%) |

## 6. ONNX Runtime 가속 (신규)

### 6.1 구성

| 모델 | src | dst | imgsz | dynamic batch |
|------|-----|-----|------:|:-:|
| YOLO11n (det) | `yolo11n.pt` | `yolo11n.onnx` | 640 | ✓ |
| v8lb R12 | `round_12_v8lb_best.pt` | `round_12_v8lb.onnx` | 128 | ✓ |
| MNv4 R8 | `round_8_mnv4_best.pt` | `round_8_mnv4.onnx` | 128 | ✓ |
| **MNv4 R15** | `round_15_mnv4_balanced.pt` | `round_15_mnv4.onnx` | 192 | ✓ |

### 6.2 구현

- **`export_onnx.py`** (신규): skip-if-exists 가드, 1회 export 후 재사용
- **`ensemble_classifier.py`**: `.onnx` 파일 자동 검출 → onnxruntime 우선 사용
- **Fallback**: ONNX 로드 실패 시 PyTorch/Ultralytics

### 6.3 실측 (batch=8)

| 엔진 | 1 iter | 평균 (10 iter) | 비고 |
|------|------:|------:|------|
| PyTorch (v2) | 358 crops/sec | 2.79 ms/crop | 기존 |
| **ONNX (v3)** | ~500+ crops/sec | **~2.0 ms/crop** | 영상 단위 50s/2000 frames (40 fps/worker) |

### 6.4 배치 가속 효과

- 3 worker × ONNX = **~120 영상 fps** 처리 가능
- 2,581 영상 × ~5,400 프레임 / 120 fps = **~32 시간 → ~11 시간** (3× 가속)

## 9. 핵심 발견 (v2 → v3)

### 9.1 Vehicle-level 집계는 정확
- Frame 62.9%에도 T1 오차 **+0.1%p** (11.3 frame 다수결로 정답 수렴)
- Urban Engineer 가설 실증 → **Frame 최적화 과집착에서 Vehicle-level 실무 현실로 전환**

### 9.2 Domain gap 53 → 12.5%p
- R14 MNv4 통합으로 multi-domain capability 획득
- R15 확장으로 양 도메인 동시 개선 (rare event)

### 9.3 Tracker class consistency 93.9%
- 74% track이 flip 0 (완전 안정)

### 9.4 뷰 이슈 해소 실증
- 북대전 정면/후면 뷰에서 100% 안정

### 9.5 원거리 30% 오분류는 area × conf 가중 다수결로 실효 해결
- T1 0.1%p 오차로 증거

### 9.6 Lazy Classification으로 합류부 구조적 문제 해결
- 물리적 한계 (track span < classify_every_n)를 알고리즘으로 우회
