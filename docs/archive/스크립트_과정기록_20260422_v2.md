## 1. 신규/대폭 변경 스크립트 (v2)

### 1.1 `export_onnx.py` 🆕

- **목적**: Ultralytics / timm 모델을 ONNX로 1회 export → `output/_models_preserved/onnx/` 재사용
- **방법**:
  - Ultralytics (`yolo11n`, `v8lb R12`): `YOLO.export(format='onnx', dynamic=True, simplify=True)`
  - timm (`MNv4 R8`, `MNv4 R15`): `torch.onnx.export(opset=17, dynamic_axes={0:'batch'})`
- **가드**: `dst.exists()` → skip (재실행 안전)
- **파라미터**:
  - `yolo11n.onnx`: 640px, dynamic batch
  - `round_12_v8lb.onnx`: 128px, dynamic
  - `round_8_mnv4.onnx`: 128px, dynamic
  - `round_15_mnv4.onnx`: 192px, dynamic
- **결과**: 4 파일, 총 ~37MB (.data 포함)
- **판단근거**: 사용자 요청 "B로 가는데 매번 export 안하게끔만" — 1회 export 후 파이프라인 자동 재사용

### 1.2 `video_consistency.py` (완전 재작성) 🔄

- **목적**: v1의 `classify_every_n=10` 합류부 95% 누락 문제 해결
- **방법 (Lazy Classification)**:
  1. 전 프레임 detector + tracker 유지 (탐지 무결성 확보)
  2. track별 crop buffer에 quality score 기록 (area × conf × center_proximity)
  3. Virtual Line Crossing (y=H/2) 기록만 (IC 각도 다양성)
  4. 영상 종료 후 필터링: `min_track_frames≥6` AND `min_trajectory≥80px`
  5. 품질 상위 **best-N=3 crops** 선별 → batch classify
  6. conf-weighted majority vote
- **파라미터**:
  - `best_n_crops=3`, `min_trajectory_px=80`, `min_track_frames=6`
  - `use_tta=False` (배치 속도 우선)
- **결과 (hiv00179 합류부)**:
  - valid tracks: 5 → **177 (35×)**
  - 분류 호출량: 프레임당 → track당 (-99%)
- **판단근거**: 5스킬 회의 (ML+Vision+Urban+Architect+QA) Option A 합의

### 1.3 `batch_all_ic.py` (개선) 🔄

- **목적**: 23 IC × 2,581 영상 전체 배치 + CPU 2,200% 활용
- **개선점**:
  - `ProcessPoolExecutor(initializer=_worker_init)` — 앙상블 worker당 1회 로드 (v1은 영상당 로드)
  - `N_WORKERS=3`, `OMP_NUM_THREADS=6`, `torch.set_num_threads(6)` → ~2,200% CPU
  - `classify_every_n=5` (v1 10→5, 하지만 v2에서는 Lazy로 사실상 무시)
  - `max_frames=None` (v1 4000→전체)
- **결과**: 2026-04-22 16:39 시작 (PID 905611), ETA ~11h
- **판단근거**: 사용자 "앙상블이 계속 로드되는데 다시 호출 안하게 하자" + "cpu 1800% 쓰도록" + "왜 4000프레임까지밖에 안하지"

### 1.4 `ensemble_classifier.py` (개선) 🔄

- **추가**: ONNX Runtime 자동 감지 (`_ort` sentinel)
- **경로**: `.onnx` 존재 시 `ort.InferenceSession` 우선, 실패 시 PyTorch fallback
- **헬퍼**: `_preprocess_rgb_chw()`, `_softmax()` 공통화
- **버그 픽스 (본 세션)**: ONNX 초기화 early-return 시 `self.mnv4_r8` 속성 누락 → `AttributeError`
  - **픽스**: ONNX 경로에서 `self.mnv4_r8 = self.mnv4_r8_sess` (sentinel)
- **실측 (batch=8)**: ONNX ~2.0ms/crop (PyTorch 2.79ms 대비 ~30% 가속)

### 1.5 학습 스크립트 (R15 산출)

- **`train_mnv4_refit.py`**: R14 base → 확장 자체 라벨(+870) → 25ep fine-tune
- **파라미터**: imgsz=192, weighted CE, lr backbone×0.1, head×1.0
- **결과**: val 71.4% @ ep19, 봉인 HW 64.6 / URB 77.3 (양 도메인 동시 개선)
- **아티팩트**: `output/_models_preserved/round_15_mnv4_balanced.pt`

## 2. 현재 활성 파이프라인 (v2)

```
┌── Stage A: 검출 (YOLO11n ONNX, 640px)
├── Stage B: 추적 (ByteTrack)
├── Stage C: Crop 축적 (track buffer + quality score)
├── Stage D (NEW): Lazy Batch Classification
│     ├── 필터 (trajectory + min_frames)
│     ├── Top-3 quality crops
│     └── Triple Ensemble ONNX (R12 v8lb + R8 MNv4 + R15 MNv4)
└── Stage E: conf-weighted vote → CSV + ic_summary.json
```
