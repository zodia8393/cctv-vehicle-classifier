## 6. 핵심 발견

### 6.1 val 99.2%는 허상

```
val 99.2%: 학습 데이터와 같은 분포에서 자기 평가 → 자기 일관성
GT 75.4%:  독립 홀드아웃에서 실제 평가 → 진짜 성능

갭 23.8%p의 원인:
  40%: 학습/평가 데이터 순환 (같은 자동 라벨)
  25%: 홀드아웃 난이도 편향 (층화추출)
  20%: 클래스 불균형 (T1 70% 지배)
  15%: 128×128 리사이즈 + 촬영 각도
```

### 6.2 T3는 학습 클래스가 아니라 후처리 규칙

- DINOv2 C3 15,405장: 99.3%가 T1 오라벨 → 폐기
- 영상 수집 T3 2,512장: 순도 40% → 7종 학습 시 catch-all화 → 실패
- **확정**: 6종 모델 + bbox 크기 기반 T3 생성 (현행 방식 유지)

### 6.3 자동 재라벨링은 위험

- conf≥0.8 자동 정제 682장 → GT 75.4%→73.7% (-1.7%p) **악화**
- 수동 검수 없는 자동 정제 = confirmation bias

### 6.4 측면 우선 crop은 효과적

- 최대 bbox → 측면 복합 점수 (AR 2.0~3.0 우선)
- T1 비율: 72.1% → 79.8% (+7.7%p)
- 상면(지붕) crop 제거로 분류 정보량 증가
