### 4.4 ID 스위칭 상세

```
변동 패턴 TOP 5:
  T1↔T3:        15,619건 (48.5%)  ← bbox area 15,000 경계 왕복
  T1↔UNK:        5,661건 (17.6%)  ← 초기 UNK → 분류 확정
  T1↔T4:         4,233건 (13.1%)  ← 원근에 따른 크기 변동
  T1↔T3↔T4:      2,184건 ( 6.8%)
  T1↔T2:           865건 ( 2.7%)
```

**근본 원인**: `_classify_with_hybrid()` 함수가 현재 프레임의 bbox 면적으로 매 프레임 재분류. 차량이 원근 이동하며 면적이 임계값(15,000)을 넘나들 때 T1/T3 전환 발생.

**해소 방안**: 집계 시 tracker_id별 최빈값(mode) 후처리 적용으로 사실상 해소됨 (파이프라인 재처리 불필요).

## 5. 파이프라인 기술 사양

### 5.1 처리 파이프라인 (하이브리드 v3.2)

```
AVI 영상 (10fps, 1920×1080)
  ↓
[1] 야간 자동 보정 (CLAHE, 평균 조도 < 110)
  ↓
[2] YOLO 검출 (yolov8n, imgsz=1280, conf=0.15)
     car(2), motorcycle(3), bus(5), truck(7)
  ↓
[3] ByteTrack 추적 (lost_track_buffer=60, match_thresh=0.7)
  ↓
[4] 하이브리드 분류 (cls_v5_best.pt + 5중 방어)
     ① 화물차 전면 방어: COCO truck + 종횡비/면적 기반 T3/T4/T5 보정
     ② 오토바이 방어: COCO motorcycle → T13 강제
     ③ 버스 오판 방어: COCO bus + 승용 오판 → T2 환원
     ④ 노이즈 방어: 30px 미만 bbox 종횡비 기록 제외
     ⑤ UNK 폴백: 3초 대기 후 COCO 기반 크기별 분류
  ↓
[출력] MP4 시각화 영상 + CSV 로그
```
