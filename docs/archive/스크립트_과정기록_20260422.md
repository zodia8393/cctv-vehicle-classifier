## 1. 신규/대폭 변경 스크립트 (v2 → v3, 04-20 이후)

### 1.1 `export_onnx.py` 🆕

| 항목 | 내용 |
|------|------|
| **목적** | Ultralytics / timm 모델을 ONNX로 1회 export → `output/_models_preserved/onnx/` 재사용 |
| **방법** | Ultralytics(`yolo11n`, `v8lb R12`): `YOLO.export(format='onnx', dynamic=True, simplify=True)`. timm(`MNv4 R8/R15`): `torch.onnx.export(opset=17, dynamic_axes={0:'batch'})` |
| **파라미터** | 각 모델별 `dst.exists()` skip 가드. yolo11n=640, R12=128, R8=128, R15=192. 모두 dynamic batch |
| **결과** | 4 파일, ~37MB (.data 포함) |
| **판단근거** | 사용자 요청 "B로 가는데 매번 export 안하게끔만" — 1회 export 후 재사용으로 배치 시작 시간 단축 |

### 1.2 `video_consistency.py` (완전 재작성)

| 항목 | 내용 |
|------|------|
| **목적** | v1의 `classify_every_n=10` 합류부 95% 누락 문제 해결 |
| **방법** | Lazy Classification: 전 프레임 track + crop buffer → 영상 종료 시 best-N batch classify + conf-weighted vote. Virtual Line Crossing 기록 (FHWA/VDS 참조) |
| **파라미터** | `best_n_crops=3`, `min_trajectory_px=80`, `min_track_frames=6`, `virtual_line_y=frame_h/2`, `use_tta=False` (배치 속도 우선) |
| **결과** | hiv00179 합류부 valid tracks: **5 → 177 (35×)**. 분류 호출 -99% |
| **판단근거** | 5스킬 회의 (ML+Vision+Urban+Architect+QA) Option A 합의. 물리적 한계(track span 12-21 frames < classify_every_n=10)를 알고리즘으로 우회 |

### 1.3 `batch_all_ic.py` (개선)

| 항목 | 내용 |
|------|------|
| **목적** | 23 IC × 2,581 영상 전체 배치 + CPU 2,200% 활용 |
| **개선점** | `ProcessPoolExecutor(initializer=_worker_init)` — 앙상블 worker당 1회 로드. `N_WORKERS=3`, `OMP_NUM_THREADS=6`, `torch.set_num_threads(6)`. `max_frames=None` (전체 영상) |
| **파라미터** | `classify_every_n=5` (Lazy로 사실상 무시), `use_tta=False`, `min_lived_frames=5`(하위호환) |
| **결과** | 2026-04-22 16:39 시작 (PID 905611), ETA ~11h |
| **판단근거** | 사용자 "앙상블이 계속 로드되는데 다시 호출 안하게 하자" + "cpu 1800% 쓰도록" + "왜 4000프레임까지밖에 안하지" |

### 1.4 `ensemble_classifier.py` (개선)

| 항목 | 내용 |
|------|------|
| **목적** | Triple Ensemble + ONNX Runtime 자동 전환 |
| **구성** | R12 v8lb(128) + R8 MNv4(128) + R15 MNv4(192), weights=(0.34/0.33/0.33) |
| **추가** | ONNX Runtime 자동 감지(`_ort` sentinel). `.onnx` 존재 시 `ort.InferenceSession` 우선, 실패 시 PyTorch fallback. TTA 4 views (center, hflip, TL crop, BR crop) |
| **버그 픽스 (본 세션)** | ONNX early-return 시 `self.mnv4_r8` 속성 누락 → `AttributeError`. **픽스**: `self.mnv4_r8 = self.mnv4_r8_sess` (sentinel) |
| **실측 (batch=8)** | ONNX ~2.0ms/crop (PyTorch 2.79ms 대비 ~30% 가속), 500+ crops/sec |

## 4. 파이프라인 Phase별 스크립트 매핑 (히스토리)

```
Phase 0: 검출 검증
  → detector.py (YOLO11n + ONNX)

Phase 1: 트래킹 + 카운팅
  → tracker.py (ByteTrack)
  → video_consistency.py (v2: Lazy Classification) ⭐ 현행
  → [legacy] render_tracking_video.py + run_batch_all_crops.py

Phase 2: 분류기 학습 (역사적)
  ├ v1 시기: extract_embeddings → cluster → finalize → confident_learning → self_training
  │          → build_dataset_v5 → train_v5 (→ cls_v5_best.pt, val 99.3%)
  ├ v2 시기: v8lb_self (Letterbox+CLAHE+self)
  └ v3 시기: ⭐ train_mnv4_full.py / train_mnv4_refit.py (→ R8, R12, R14, R15)

Phase 3: 평가
  → eval_holdout.py (Bootstrap CI + McNemar + multi-domain)
  → 봉인 v3: main_test_v3_manifest.jsonl + urban_blind_test_manifest.jsonl

Phase 4: 배치 처리
  → batch_all_ic.py (ProcessPoolExecutor + ONNX + 3 worker) ⭐ 현행
  → [legacy] massive_batch_process.py (부평IC 93영상)

Phase 5: 집계 + 보고
  → analyze_traffic.py (집계)
  → generate_report.py / generate_refined_report.py (시각화)
  → ic_summary.json + weak_spots.md (자동 생성)

배포:
  → export_onnx.py (1회 export, skip-if-exists) ⭐ NEW
```

## 8. 카테고리별 상세 과정 (핵심만 — 전체는 archive/ 참조)

### 8.1 데이터 준비 & 클러스터링 (Phase 2 초기)
- **목적**: 147K crop → 차종 자동 분리
- **파이프라인**: DINOv2(384d) → UMAP(32d) → K-Means(K=50 → K=200) → 클러스터별 라벨 할당
- **핵심 상수**: K=50/200, `MIN_AREA=8000`, UMAP n_neighbors=30
- **판단근거**: GPU 없이 CPU 추론 가능. K=200은 C5(대형화물)을 세분화하기 위해.

### 8.2 라벨 정제 (v1→v5)
- v3: Confident Learning (KNN+cleanlab) — 노이즈 라벨 자동 식별
- v4: 수동 보정 (T3→T1 90%, T8→T4 0%, T5는 area≥80000만 유지)
- v5: Self-training (v4 모델 재예측, conf≥0.7 → MODEL/AGREE)
- **핵심 통찰**: GT 500장 시각 검증에서 T3(포터) 90%가 승용차 → "T3는 후처리 규칙"으로 확정

### 8.3 ByteTrack 파이프라인 (Phase 1 & 4)
- `tracker.py`: supervision.ByteTrack 래핑. TRACK_BUFFER=60, MATCH_THRESH=0.7
- best_crop: 최대 bbox 면적 기준 갱신 (가장 가까운 순간 = 최고 품질)
- **v1~v2**: `render_tracking_video.py` — classify_every_n=10, 5중 보정
- **v3 (현행)**: `video_consistency.py` — Lazy Classification (합류부 95%→해소)

### 8.4 T3 수집 (폐기된 실험)
- COCO truck + CLS T1 교차 패턴으로 포터/봉고 수집 시도
- 1,504장 수집 (`massive_extraction.py`) + 캐싱 (`label_and_cache.py`)
- **결과**: 학습 시 catch-all 되어 실패 → 후처리 규칙으로 전환

### 8.5 Active Learning (1회 완료)
- `active_learning.py`: run_and_cache → review_grid → apply_corrections → retrain
- retrain lr=0.0003 AdamW (기존 지식 보존하며 미세 조정)
- **결과**: Round 1에서 GT 악화 발견, 자동 재라벨링 위험성 확인 → 수동 검수 원칙 확립
