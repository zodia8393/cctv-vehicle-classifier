## 1. 실험 배경

### 1.1 현행 시스템

| 항목 | 내용 |
|------|------|
| 파이프라인 | YOLOv8n 검출 → ByteTrack 추적 → cls_v5 분류 → 하이브리드 5중 보정 |
| 코드 규모 | 740줄, 모델 2개, 외부 래퍼 3개, 보정 로직 5개 |
| 분류 모델 | YOLOv8n-cls (2.72M params, 2.9MB, val 99.3%) |
| 분류 체계 | 6종 (T1/T2/T4/T5/T10/T13) + 하이브리드 보정으로 T3 추가 = 실질 7종 |
| 부평IC 결과 | 93영상 처리 완료, QA 조건부 PASS |
| 주요 문제 | T1↔T3 스위칭 14.3%, UNK 8.8%, 코드 복잡도 높음 |

### 1.2 실험 목표

1. **분류 모델 업그레이드**: YOLOv8n-cls → YOLO11n-cls (최신 아키텍처)
2. **파이프라인 단순화**: 검출+분류 통합 모델 (YOLOE / YOLO-World) 검토
3. **12종 확장 가능성**: 도로교통량 조사지침 12종 분류 달성 방안

## 2. 실험 1: YOLO11n-cls 업그레이드

### 2.1 목적

기존 YOLOv8n-cls(val 99.3%)를 YOLO11n-cls로 교체하여 동등 이상 성능 달성

### 2.2 아키텍처 비교

| | YOLOv8n-cls | YOLO11n-cls |
|---|---|---|
| 백본 블록 | C2f | **C3k2** (커널 크기 가변) |
| 어텐션 | 없음 | **C2PSA** (공간 어텐션) |
| 파라미터 | 2.72M | **1.54M (43% 감소)** |
| ImageNet Top-1 | 69.0% | 70.0% |

### 2.3 실험 과정

#### 2.3.1 심링크 깨짐 이슈 (사전 해결)

- dataset_v5 심링크 38,696개가 경로 변경(`CCTV차종분류_output` → `CCTV차종분류/output`)으로 전부 깨짐
- Python으로 일괄 수정 완료
- **교훈**: 학습 전 `find -type l ! -exec test -e {} \;`로 심링크 검증 필수

#### 2.3.2 1차 학습 (ImageNet → dataset_v5)

| 설정 | 값 |
|------|------|
| 베이스 | yolo11n-cls.pt (ImageNet pretrained) |
| 데이터 | dataset_v5 (32,895 train / 5,801 val, 6종) |
| 설정 | epochs=30, imgsz=128, batch=256, workers=16, AdamW, lr=0.001, cosine LR |
| 소요 | 45분 |

**결과**: val Top-1 **96.7%** (30 epochs 완주)

#### 2.3.3 Self-training (v8 Teacher → v11 Student)

| 단계 | 내용 | 결과 |
|------|------|------|
| Teacher 예측 | cls_v5(99.3%)로 147,521장 전체 예측 | 133,286장 통과 (conf≥0.9, 90.4%) |
| 데이터셋 구성 | 133K장, 90/10 train/val 분할 | 6종 × 심링크 |
| Student 학습 | v11 1차(96.7%) warm start, lr=0.0005, epochs=20 | **val 99.2% (Epoch 2에서 조기종료)** |
| 소요 | Teacher 예측 149초 + 학습 ~15분 | |

### 2.4 실영상 검증 (hiv00220)

| 차종 | v8 (기존) | v11 | 차이 |
|------|----------|-----|------|
| T1 승용차 | 71.1% | 72.3% | +1.2%p |
| T2 버스 | 6.1% | 10.1% | +3.9%p |
| T4 중형화물 | 19.7% | 14.5% | -5.1%p |
| T5 대형화물 | 0.6% | 0.7% | +0.1%p |
| UNK | 16.8% | 16.8% | 0 |
| **총 차량** | **2,834** | **2,834** | **0 (완전 동일)** |

- 검출/추적은 동일 → 총 차량 수, UNK 완전 동일
- T2/T4 경계에서 약 90대 차이 (C2PSA 어텐션이 측면 패널→버스 창문 해석)
- **결론: 교체 가능 (동등 성능)**

### 2.5 최종 결과

| 지표 | v8 (기존) | v11 self-trained |
|------|----------|-----------------|
| Val Top-1 | 99.3% | **99.2%** |
| 파라미터 | 2.72M | **1.54M (43%↓)** |
| 모델 크기 | 2.9MB | 3.1MB |
| 추론 속도 | 0.3ms | 0.4ms |

**판정: 교체 가능.** 파라미터 43% 감소 + 최신 아키텍처 확보. 모델 위치: `training_v11/self_training/weights/best.pt`

## 3. 실험 2: YOLOE Zero-shot PoC

### 3.1 목적

YOLOE(open-vocabulary detection)의 텍스트 프롬프트로 12종+특수차량 분류 가능성 검증

### 3.2 모델

- YOLOE-11L-seg (35.2M params, 67.7MB)
- CLIP 텍스트 인코더 기반 open-vocabulary 검출

### 3.3 프롬프트 진화

| 버전 | 클래스 수 | 승용차 비율 | 경찰차 오탐 | 결과 |
|------|---------|----------|----------|------|
| v1 (17종, 단순 영어) | 17 | 24.0% | **42.0%** | 실패 |
| v2 (17종, 정밀 묘사) | 17 | 58.5% | 2.7% | 개선 |
| v3 (17종, 극정밀) | 17 | 51.6% | 0.7% | 특수차량 해소, T5/T9 혼동 |
| v4 (13종, 특수차량 제외) | 13 | 54.1% | - | 기대 78% 미달 |

### 3.4 근본 한계

- CLIP 텍스트 임베딩에서 "medium truck"과 "large truck"의 거리가 시각적 차이보다 작음
- 유사 클래스 간 구분 불가 (T4/T5/T6/T7)
- **결론: Zero-shot 텍스트 프롬프트만으로 12종 세분류 90%+ 불가능**

### 3.5 추론 속도

- 0.13초/프레임 (CPU, 1920×1080, 17종 기준)
- 기존 파이프라인(검출+분류 합산)과 동등

## 5. 실험 4: YOLO-World Fine-tune

### 5.1 목적

YOLO-World를 우리 데이터로 fine-tune하여 검출+분류 통합 파이프라인 구축

### 5.2 데이터셋 구축

| 항목 | 값 |
|------|------|
| 원본 | 부평IC 93영상 트래킹 CSV (161,323 bbox) |
| 추출 방식 | 30프레임 간격 샘플링 → 풀프레임 + YOLO format bbox 라벨 |
| 프레임 수 | 24,285장 (train 18,920 / val 5,365) |
| bbox 수 | 161,323개 (7종) |
| 클래스 분포 | T1 83.1%, T4 11.1%, T2 2.6%, T3 1.5%, T5 1.5%, T10 0.1%, T13 0.1% |

### 5.3 학습 설정

| 설정 | 값 |
|------|------|
| 모델 | yolov8s-worldv2.pt (Small) |
| imgsz | 640 |
| batch | 16 → 32 (cache=ram 적용 후) |
| workers | 16 (CPU 70%) |
| cache | 없음 → ram (Epoch 4~) |
| optimizer | AdamW, lr=0.001, cosine LR |
| patience | 5 |

### 5.4 학습 결과 (17 Epochs, 중단)

| Epoch | box_loss | cls_loss | Precision | Recall | mAP50 | mAP50-95 |
|-------|----------|----------|-----------|--------|-------|----------|
| 2 | 0.897 | 1.090 | 0.587 | 0.241 | 0.261 | 0.201 |
| 4 | 0.824 | 0.950 | 0.439 | 0.380 | 0.308 | 0.249 |
| 7 | 0.750 | 0.845 | 0.321 | 0.399 | 0.341 | 0.277 |
| 10 | 0.710 | 0.797 | 0.354 | 0.350 | 0.331 | 0.273 |
| **13** | **0.692** | **0.770** | **0.308** | **0.505** | **0.360** | **0.298** |
| 16 | 0.667 | 0.734 | 0.287 | 0.487 | 0.359 | 0.301 |
| 17 | 0.667 | 0.734 | 0.287 | 0.487 | 0.359 | 0.301 |

### 5.5 분석

- **mAP50**: 0.261 → 0.360 (Epoch 13 best), 이후 정체
- **Precision**: 0.587 → 0.287 (지속 하락) — 오탐 증가
- **Recall**: 0.241 → 0.505 (지속 상승) — 검출 누락 감소
- **Loss**: 꾸준히 감소하나 mAP 미반영 → 과적합 진입 징후

### 5.6 실패 원인 분석

| 원인 | 상세 |
|------|------|
| **라벨 노이즈** | 기존 파이프라인의 자동 라벨 사용 (T1↔T3 스위칭 14%, UNK 8.8%) |
| **클래스 극심한 불균형** | T1 83% vs T10/T13 0.1% (833배 차이) |
| **데이터 부족** | 검출 모델은 수만~수십만 장 필요, 현재 19K장 |
| **태스크 난이도 차이** | crop 분류(128×128, 6종)와 풀프레임 검출(640×640, 7종)은 근본적으로 다른 문제 |
| **imgsz 제한** | 640 학습으로 원거리 소형 차량 bbox(10~20px) 학습 부족 |

### 5.7 판정

**NO-GO.** mAP50 0.36은 GO 기준(0.85)에 크게 미달. 현 조건에서 추가 학습해도 0.40 수준이 한계.

### 5.8 모델 파일

- best.pt (74MB, Epoch 13 기준): `training_yoloworld/finetune_v1/weights/best.pt`
- 향후 조건 충족 시 warm start로 활용 가능

## 7. 인프라 실험: 학습 최적화

### 7.1 CPU 병렬화 (workers=16)

- 24코어 중 16코어(70%) 할당
- 데이터 로딩 병목 해소, 학습 루프 I/O 대기 제거

### 7.2 RAM 캐시 (cache="ram")

| | cache 없음 | cache="ram" |
|---|---|---|
| RAM 사용 | 8GB | 29GB |
| epoch당 | ~70분 | ~45분 (추정 36% 단축) |
| 효과 | 매 epoch 디스크 읽기 | I/O 제로 |

- 122GB RAM 환경에서 학습 데이터(~30GB)를 전부 RAM에 캐싱
- 단, 다른 대형 프로세스(부천 분석 57GB)와 동시 실행 시 OOM 위험

### 7.3 tmpfs 가속

- 영상 파일을 `/dev/shm` (RAM 디스크)에 복사 후 처리
- 외장 HDD 250MB/s → RAM 50,000MB/s (200배)
- IC 1개 분량(~18GB)씩 순환 복사 → 배치 처리 I/O 병목 제거

## 13. 실험 5: Adaptive Conf + ROI Upscale (진행 중)

### 13.1 목적

v11-cls 확정 후, 원거리 차량(40px 미만) 검출률을 모델 교체 없이 개선

### 13.2 배경 — ML Engineer 실측 발견

| 설정 | 총 검출 | 40px 미만 | 비고 |
|------|--------|----------|------|
| conf=0.15 (현행) | 74대 | 7대 | 기존 |
| conf=0.05 (전부 수용) | 280대 | 57대 | 오탐 과다 |
| **Adaptive 필터** | **91대** | **24대** | **오탐 없이 3.4배 개선** |

**핵심 발견**: 모델(YOLOv8n)은 이미 conf=0.05에서 소형 차량을 검출하고 있었다. conf=0.15 고정 임계값이 소형 객체를 차단하고 있던 것이 문제의 본질.

### 13.3 적용 알고리즘

#### A. Adaptive Conf Threshold (bbox 크기별 차등 필터링)

```python
# conf=0.05로 전부 받은 뒤, 크기별 차등 적용
if size >= 40px and conf >= 0.15:   # 중대형: 현행 유지 (오탐 방지)
if size >= 20px and conf >= 0.08:   # 소형: 우대
if size >= 10px and conf >= 0.05:   # 극소형: 최대 수용
```

- 속도 영향: **0ms** (후처리 필터링만)
- ByteTrack이 단발성 오탐(1~2프레임) 자동 소멸

#### B. ROI 원거리 2x Upscale

```python
# 화면 상단 37% (원거리 영역)만 2배 확대 후 재검출
roi = frame[0:int(h*0.37), :]
roi_up = cv2.resize(roi, (w*2, roi_h*2))
# → 20px 차량이 40px로 확대 → YOLO 검출 가능 범위 진입
```

- 속도 영향: **+30~40%** (소형 crop 추론 1회 추가)
- 전체+ROI 결과를 NMS 병합하여 중복 제거
